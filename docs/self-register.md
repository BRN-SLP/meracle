# Registering meRacle on the Self Agent ID Registry

The Self Agent ID is the second leg of the AI Agents Prize Pool
requirement (alongside the ERC-8004 Identity NFT). The flow runs
through the public Self HTTP API in `wallet-free` mode and the Self
mobile app on a phone with a passport-verified user.

## Where it lives

| Field | Value |
|---|---|
| Self HTTP API | `https://app.ai.self.xyz/api/agent` |
| Self Agent ID Registry | `0xaC3DF9ABf80d0F5c020C06B04Cced27763355944` (Celo Mainnet) |
| Script | `scripts/register-self.ts` |
| Trigger | `pnpm register:self` |
| meRacle Self Agent ID | `119` |

## Architecture (three identities, linked off chain)

`mode: "wallet-free"` keeps the script trivial and avoids the broken
`ed25519-linked` path. As a result the meRacle on-chain stack runs on
three separate addresses, linked together through this repo's public
`agent.json`:

| Role | Address | Owns |
|---|---|---|
| Operational hot wallet | `0x1B94d56f723d8939661D94eD1f899C5c27136b2c` | ERC-8004 Identity NFT, signs `submitPrice()` |
| Self-managed wallet | `0xE91Fd18BB043338D81e3C4c75100a4799DFfF299` | Self Agent ID NFT `#119` (passport ZK proof) |
| Personal wallet | operator's own EVM account | Optional, not in scope for the bot itself |

The hot wallet is the only one that spends gas. The Self-managed
wallet was generated by Self at session start and exists purely to
hold the soulbound Agent ID NFT.

## Prerequisites

- A phone with the Self mobile app installed.
- A passport already verified in Self (the operator can see it under
  "Manage ID" in the app).
- `pnpm` and Node `>=20`.
- `.env` set up per `.env.example` (no extra fields needed for this
  flow, `AGENT_PRIVATE_KEY` is enough).

## Run

```bash
pnpm register:self
```

The script does the following on Celo Mainnet:

1. `POST /api/agent/register {mode: "wallet-free", network: "mainnet"}`
2. Prints a deep link the operator opens on their phone (the Self
   mobile app picks it up automatically) and a scan URL for the
   desktop QR fallback.
3. The operator approves and taps the passport (NFC), Self generates
   the ZK proof and submits the on-chain `register` transaction.
4. The script polls `GET /api/agent/register/status` every 3 seconds
   with `Authorization: Bearer <sessionToken>` until the stage is
   `completed` (or `registered`, depending on the variant).
5. Prints the resulting `agentId`, `txHash`, and the Celoscan URL.

The session has a 30 minute time window. A rerun just opens a fresh
session, the previous one expires harmlessly.

## Verifying the entry on chain

The Registry follows ERC-721, so `ownerOf(tokenId)` gives the address
holding the NFT. For Agent ID `119`:

```bash
cast call --rpc-url https://forno.celo.org \
  0xaC3DF9ABf80d0F5c020C06B04Cced27763355944 \
  "ownerOf(uint256)(address)" 119
# -> 0xE91Fd18BB043338D81e3C4c75100a4799DFfF299
```

## Prize pool checklist after this step

When all three items are present, the meRacle entry qualifies under
the AI Agents Prize Pool rules:

- [x] ERC-8004 Identity NFT minted (`pnpm register:identity`, on
      `0x1B94d56f723d8939661D94eD1f899C5c27136b2c`)
- [x] Self Agent ID registered through this flow (Agent ID `119`, on
      `0xE91Fd18BB043338D81e3C4c75100a4799DFfF299`)
- [x] On-chain transaction history from the operational wallet (the
      daily cron submits via `submitPrice()` to the Mercato Reference
      Price Oracle)

## Notes on the `ed25519-linked` attempt

The intent was to mint the Self Agent ID NFT directly on a personal
EVM wallet (`humanAddress` parameter on `mode: "ed25519-linked"`).
Five passport scans across five disclosures configurations all
surfaced "Proof Failed, technical issue" in the Self app with the
session never advancing past `qr-ready`. The history is preserved
in the `phase-0/self-register-script` branch so the path can be
revisited once Self ships a fix or documentation lands.

## If anything is off

The Self HTTP API is moving quickly. If the script breaks on a
schema mismatch, run a single API call by hand to see the current
payload and update the matching `interface` in
`scripts/register-self.ts`, then commit the diff with a `fix(...)`
or `feat(...)` message that names the field that changed.
